Part 1: Visual Effect Idioms Catalog

4 Categories of LED Effects

Effects organized by their relationship to physical form. Each category exploits different properties of sculpture topology.

1. Strip-Native
Proven 1D LED effects from WLED/FastLED. Motion, noise, particles, waves, composition.
2. Topology-Aware
Exploit physical form: hub-and-spoke, height-mapped, symmetry. The signature effects.
3. Shader/VJ Adapted
2D effects as 1D slices. Metaballs, reaction-diffusion, cellular automata, flow fields.
4. Nature Simulations
Physical presence effects. Candle flicker, bioluminescence, electrical storms, breathing.

Part 2: Automated Show Architecture

Multi-Hour Show Control Without a VJ

VJs make three real-time decisions: effect selection, parameter modulation, transitions. Automate the first two, prepare the third.

Detection Stack
4 layers: frame-level, beat-level, phrase-level, song-level. Phrase-level is the critical gap.
Show Controller
Section detector, effect selector, parameter modulator, palette manager. Orchestrates the show.
Section Mapping
Intro/verse/build/drop/breakdown/climax/outro → effect families with configurable defaults.
Fatigue Prevention
Effect rotation, palette cycling, complexity curve, spatial rotation, novelty injection.
Detection Stack (4 Layers)
Layer 1: Frame-Level (Already Have)
Real-time audio features computed every frame (30 FPS). Foundation for all higher-level detection.
  • RMS / loudness — envelope tracking
  • Onset / transient — per-band spectral flux
  • Spectral centroid — brightness/warmth
  • Per-band energy — 5-band mel decomposition
  • Absint (rate of change) — derivatives over absolutes
Layer 2: Beat-Level (Partially Have)
Tempo and rhythm tracking. Octave-ambiguous but functional for most use cases.
  • Tempo (BPM) — implemented, octave-ambiguous
  • Beat prediction — BeatPredictor in signals.py
  • Onset envelope — OnsetTempoTracker
  • Downbeat detection — partial (every-4th heuristic, not true downbeat)
  • Vocal presence — NOT implemented (spectral shape heuristics or ML)
Layer 3: Phrase-Level (Critical Gap)
Where automated show control lives. Largely unimplemented. Highest priority.
  • Build detection — rolling integral slope > threshold for N seconds (medium difficulty)
  • Drop detection — RMS jump after build phase, bass energy spike (medium)
  • Breakdown detection — rolling integral slope < 0, spectral flatness drops (medium)
  • Section boundary — self-similarity novelty, streaming alternative to Foote (hard)
  • Riser detection — rising spectral centroid + increasing RMS over 2-8s (medium)
  • Dropout detection — per-band energy drops to near-zero (easy — already have normalization)
Layer 4: Song-Level (Future / Optional)
Long-horizon adaptation for multi-hour sets. Low priority until phrase-level works.
  • Song transition — one track fading into another (DJ context)
  • Genre/mood shift — sliding window over spectral shape, onset density
  • Energy arc — set-wide intensity trajectory (prevent visual fatigue over hours)
  • Key detection — chroma analysis for palette selection (feasible but low priority)

Priority: Build detection → drop detection → breakdown → section boundary. These four cover 80% of automated show control decisions.

Show Controller Components
Section Detector
Consumes frame-level features. Outputs build/drop/break state and section boundaries. Streaming alternative to Foote's method.
Effect Selector
Section → effect map with transition logic and fatigue prevention. After 3 drops with Fire, switch to Root Pulse.
Parameter Modulator
Audio → visual mapping. Continuous modulation of effect intensity, speed, color, complexity from audio features.
Palette Manager
Mood → palette map with rotation timer. Cycles through warm, cool, monochrome, complementary families over 10-20 min periods.

Visual Fatigue Prevention

Strategies for Multi-Hour Autonomy

For multi-hour automated shows, the system needs strategies to stay interesting and avoid visual habituation.

1. Effect Rotation
Don't use the same effect for more than N consecutive sections of the same type. After 3 drops with Fire, switch to Root Pulse or Electrical Storm.
2. Palette Cycling
Rotate through palette families (warm, cool, monochrome, complementary) over 10-20 minute periods, independent of section detection.
3. Complexity Curve
Track a long-horizon "visual complexity" metric. If it's been high for too long, force a simpler section. Mirror the concept of dynamic range in audio mastering.
4. Spatial Rotation
Alternate between topology modes (full-strip, zoned, height-mapped, branch-independent) to keep the spatial experience fresh.
5. Novelty Injection
Periodically introduce a rare effect (frost crystallization, reaction-diffusion) that hasn't been used recently. The surprise factor prevents habituation.

  • Manual override via controller input (human-in-the-loop for taste)
Lesson from Commercial Systems

SoundSwitch, MaestroDMX, Engine Lighting

Commercial automated lighting systems (SoundSwitch, MaestroDMX, Engine Lighting) validate the approach: phrase detection is a solved-enough problem in the commercial world. The challenge isn't detecting drops — it's making the visual response to a detected drop feel right.

This is exactly where the "two quality axes" framework applies: detection quality is necessary but mapping quality is what people see. SoundSwitch ships with 32 pre-built "Autoloops" (effect sequences for common song structures). Users customize by editing loops, not by writing detection code.
