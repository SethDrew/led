Analysis Overview
Waveform + RMS
Raw audio samples with smoothed loudness overlay. Toggle RMS with E.
Derivatives of RMS matter more than absolutes — climax brightens 58x faster than build despite identical static RMS.
Mel Spectrogram
STFT on perceptual mel scale. Bass = bright blobs at bottom, vocals = middle bands, hi-hats = top. Horizontal lines = harmonic content, vertical lines = percussive — this is why HPSS works.
Band Energy
Mel spectrogram collapsed into 5 bands: Sub-bass 20–80Hz, Bass 80–250Hz, Mids 250–2kHz, High-mids 2–6kHz, Treble 6–8kHz. Foundation of every audio-reactive LED system.
RMS Derivative Custom
Rate-of-change of loudness (dRMS/dt). Red = getting louder, blue = getting quieter.
The signal that distinguishes builds from drops. Derivatives > absolutes.
Causal Novelty Custom ESP32
Feature flux (solid): half-wave rectified frame diff. Peaks at abrupt changes — section hits, instrument entries. MFCC flux (coral) = timbral, chroma flux (teal) = harmonic.
EMA deviation (dashed): distance from exponential moving average. Peaks during gradual drift — buildups, filter sweeps.
Flux detects sharp edges, EMA detects gradual drift. Both causal, frame-by-frame, O(n).
Annotations
Tap Annotations Custom
Your tap data overlaid on analysis. Press T to tap while playing, S to save. Supports multiple layers (beat, section, flourish, etc.).
Exhibits tactus ambiguity — listeners lock onto different metrical layers per song, so taps may be phase-shifted 100–250ms (Martens 2011, London 2004).
Decomposition
Four source separation methods. Use number keys to solo/mute stems where audio playback is available.
Demucs Common
Deep learning 4-stem: drums, bass, vocals, other. ~25s CPU. Ground truth, not real-time.
HPSS ESP32
Median filtering — harmonic (horizontal) vs percussive (vertical). Frame-by-frame, trivially real-time.
REPET ESP32
Repeating pattern extraction. Autocorrelation + median filter + soft mask. Foreground vs background.
NMF ESP32
Supervised dictionary-based. 0.07ms/frame, 10KB dictionary. Most promising for real-time source attribution.
Lab
Experimental features under evaluation for LED mapping potential.
Timbral Shape
MFCC heatmap (13 coefficients). Vertical color shifts = sound character changes.
Misc
Spectral flatness (tonal vs noise), chromagram (pitch class energy), spectral contrast (peak-to-valley per band), ZCR (percussive vs smooth).
Onset + AbsInt Experimental
Side-by-side comparison. Onset = sharp on attacks. AbsInt = sticky on decay shapes. Different temporal sensitivities.
MOOD Vectors Experimental
4D mood signal: Brightness (spectral slope, warm↔cool), Texture (harmonic ratio, smooth↔rough), Tension (chroma entropy, relaxed↔tense), Fullness (spectral spread, thin↔full). Replaces single-dimensional centroid with multi-axis timbral character.
Speed Signal Experimental
Spectral evolution rate for ambient music. Blends spectral flux (multi-band energy onset), centroid rate (timbral shift speed), and chroma flux (harmonic change) into a single 0-1 speed value. Each feature peak-normalized, then averaged and EMA-smoothed (1.5s). Designed for driving rotation speed where tempo tracking fails.
Spectro-Color Experimental
Spectrum-to-LED color mapping explorer. Mel spectrogram and MFCC spectrogram with turbo colormap, plus four LED strip previews: Uniform (centroid → whole strip), Freq-Position (frequency bins → vertical position), Waterfall (scrolling history), MFCC-RGB (first 3 MFCCs → R, G, B).

Requires BlackHole 2ch for system audio capture. Recording will not work without it.

Setup: BlackHole (macOS)
  1. Install BlackHole 2ch from existential.audio/blackhole
  2. In System Settings → Sound → Output, select BlackHole 2ch
  3. Restart this server — BlackHole will be auto-detected

Tip: To hear audio while recording, open Audio MIDI Setup, click + → Create Multi-Output Device, check both your speakers and BlackHole, then set that as your system output.
