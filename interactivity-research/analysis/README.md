# Analysis

Research outputs organized by pillar.

## taste/

**Pillar 1 — Findings from user annotations.** What musical moments feel like and why, derived from tap-annotation experiments on Opiate Intro and Fred again.. tracks.

Key reports: beat_tap_analysis.md, air_feature_analysis.md, build_vs_climax.md, beat_vs_consistent.md, flourish_audio_properties.md, fa_br_drop1_analysis.md

## algorithms/

**Pillar 2 — Algorithm evaluation.** Beat tracker comparisons, library benchmarks (librosa, madmom, essentia), real-time constraint analysis, and LED mapping strategies.

Key reports: REALTIME_CONSTRAINTS.md, beat_tracker_comparison.md, library_comparison.md, LED_MAPPING_GUIDE.md

## harmonix/

**Harmonix dataset work.** Exploration, hyperparameter optimization, and validation track selection using the 912-track Harmonix Set.

Key reports: harmonix_exploration.md, harmonix_optimization.md, validation_tracks.md

## scripts/

**Reproducible analysis scripts.** Python scripts that generated the reports and data files above. Run from repo root with `venv/bin/python`.
